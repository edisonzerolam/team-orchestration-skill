# 行业咨询分析团队 SOP
## 基本信息
- **团队名称**: industry-consulting（行业咨询）
- **Agent 数量**: 25
- **触发词**: 行业研究/咨询分析/战略规划/市场调研/产业分析

## 团队架构
| Agent | 角色 | 职责 |
|-------|------|------|
| business-strategist | 商业策略师 | 战略规划、竞争分析、商业模式设计 |
| supply-chain-strategist | 供应链策略师 | 供应链分析、流程优化 |
| finance-financial-analyst | 财务分析师 | 财务分析、投资评估、成本建模 |
| operations-manager | 运营经理 | 流程设计、运营效率优化 |
| change-management-consultant | 变革管理顾问 | 组织变革、转型路径设计 |
| organizational-psychologist | 组织心理学家 | 团队动力、文化建设 |
| gis-analyst | GIS 分析师 | 地理空间数据、区域分析 |
| security-threat-intelligence-analyst | 安全情报分析师 | 行业风险分析 |
| academic-statistician | 统计分析师 | 数据分析、量化建模 |
| market-researcher | 市场研究员 | 市场规模、竞争格局、用户洞察 |（及更多 15 名专家）

## SOP 流程

### Phase 1: 调研与数据采集
**输入**: 客户需求/研究课题
**输出**: `research-brief.md`
**目的**: 明确研究范围和关键问题
**步骤**:
1. business-strategist 解析需求，确认研究框架
2. market-researcher 制定数据采集计划
3. 输出研究简报

### Phase 2: 分析与建模
**输入**: research-brief.md
**输出**: `analysis-report.md`
**目的**: 深度分析行业数据，构建分析模型
**步骤**:
1. finance-financial-analyst 进行财务建模
2. academic-statistician 量化数据分析
3. supply-chain-strategist / gis-analyst 专项分析
4. 输出分析报告

### Phase 3: 战略建议
**输入**: analysis-report.md
**输出**: `strategy-recommendation.md`
**目的**: 基于分析输出可执行的战略建议
**步骤**:
1. business-strategist 提炼核心洞察
2. change-management-consultant 评估实施路径
3. 输出战略建议文档

### Phase 4: 报告撰写
**输入**: strategy-recommendation.md
**输出**: 最终咨询报告
**目的**: 形成完整、专业的咨询交付物
**步骤**:
1. 整合所有分析结果
2. 编制执行路线图
3. 输出最终报告

## 阶段关卡（Phase Gates）
| 关卡 | 通过条件 | 失败处理 |
|------|----------|----------|
| PG1-调研 | 研究范围清晰，关键假设明确 | 返回补充调研方向 |
| PG2-分析 | 数据源可靠，分析方法合理 | 重新验证数据 |
| PG3-战略 | 建议可行，风险已识别 | 完善实施路径 |
| PG4-报告 | 逻辑严谨，格式规范 | 按模板调整 |
