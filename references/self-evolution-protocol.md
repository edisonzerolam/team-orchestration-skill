# 三环自进化协议

## 架构总览

```
Loop 1 (快速)           Loop 2 (中速)           Loop 3 (慢速)
执行中反思              事后回顾                主动联网增强
← 秒级响应 →            ← 分钟级 →              ← 小时/天级 →
                                                    
每次 subagent 执行完   整个任务完成             定时触发
→ 写 self-evolution-log → 更新 expert-scores   → 搜索 + 生成报告
→ 仅追加，不改原文      → 跨专家横评            → 用户确认后合并
```

---

## Loop 1: 执行中反思

**触发条件**: 每个 subagent 执行完毕后自动触发

**行为**:
```markdown
## 执行反思 (自动追加到 self-evolution-log.md)

- 任务: {任务简述}
- 时间: {时间戳}
- 有效方法: {什么方法/提示词效果好}
- 缺失数据: {什么数据获取不到 / 质量差}
- 改进建议: {下次如何优化}
```

**数据流**: 只写入本专家目录下的 `self-evolution-log.md`

---

## Loop 2: 事后回顾

**触发条件**: 整个多专家任务完成后

**行为**:
1. 收集所有参与专家的 self-evolution-log
2. 横评各专家表现（质量、速度、数据完整度）
3. 更新 `expert-scores.json`
4. 生成跨专家学习建议

**输出**: 更新后的评分数据库

---

## Loop 3: 主动联网增强

**触发条件**: 定时（每 N 次执行）或手动触发

**行为**:
1. 遍历所有 active 专家
2. 对每个专家按领域关键词搜索互联网
3. 生成进化报告:
   - 新发现的知识点
   - 已过时的信息
   - 可用的新工具/API
4. 用户审核后:
   - 通过 → `knowledge-merger.py` 合并到 knowledge 文件
   - 拒绝 → 丢弃

**数据流**: 半自动 — 用户确认后写入
