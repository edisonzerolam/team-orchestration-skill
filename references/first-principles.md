# 第一性原理任务拆解框架

## 核心思想

不依赖预定义模板，从任务本质出发逐层拆解，找到最原初的需求形态，再匹配合适的专家资源。

## 五维拆解

### 维度 1: 任务本质类型 (π)

| π | 类型 | 特征词 | 举例 |
|---|------|--------|------|
| π₁ | 信息检索型 | 查、找、搜索、查询、获取 | "查茅台今天股价" "找XX公司财报" |
| π₂ | 分析判断型 | 分析、评估、判断、预测、诊断 | "分析这只股票" "评估市场风险" |
| π₃ | 创作生成型 | 写、生成、设计、制作、创建 | "写一篇分析报告" "设计一个页面" |
| π₄ | 决策执行型 | 决策、执行、部署、操作 | "该不该买入" "如何配置系统" |
| π₅ | 混合型 | 以上多个组合 | "分析XX并给出交易建议" |

### 维度 2: 所需知识域

从 WorkBuddy 12 分类中匹配：

| ID | 分类 | 适用场景 |
|----|------|---------|
| 01-ProductDesign | 产品设计 | UI/UX、原型、交互 |
| 02-Engineering | 技术工程 | 编程、架构、DevOps |
| 03-GameSpatial | 游戏空间 | 游戏、3D、VR |
| 04-DataAI | 数据智能 | 数据分析、ML、BI |
| 05-MarketingGrowth | 营销增长 | 品牌、增长、广告 |
| 06-ContentCreative | 内容创作 | 文案、视频、翻译 |
| 07-SalesCommerce | 销售商务 | 销售、谈判、电商 |
| 08-FinanceInvestment | 金融投资 | 投资、财务、量化 |
| 09-OperationsHR | 运营人力 | 项目管理、HR |
| 10-ProjectQuality | 项目质量 | 质量、测试、流程 |
| 11-SecurityCompliance | 法务安全 | 合规、法务、隐私 |
| 12-IndustryConsultant | 行业顾问 | 跨行业/战略 |

### 维度 3: 所需能力

| 能力 | 说明 |
|------|------|
| 数据获取能力 | 查询行情、财报、宏观等外部数据 |
| 模式识别能力 | 识别趋势、异常、相关性 |
| 判断/决策能力 | 给出评分、评级、建议 |
| 内容生成能力 | 生成报告、代码、设计稿 |
| 执行操作能力 | 部署、交易、配置 |

### 维度 4: 复杂度

| 等级 | 特征 | 专家数 |
|------|------|--------|
| L1-简单 | 单一维度, 已知事实 | 1 |
| L2-中等 | 多维度, 需分析 | 2-4 |
| L3-复杂 | 跨领域, 需多轮推理 | 5-8 |
| L4-深度 | 需完整团队协作 | 8+ |

### 维度 5: 质量要求

| 要求 | 说明 |
|------|------|
| 精度 | approximate / exact |
| 引用 | 无 / 单源 / 多源 |
| 形式 | 对话 / Markdown / HTML / 代码 |

## 拆解流程

```
输入: 用户任务文本 T

1. 识别 π（本质类型）
2. 识别知识域（1-3 个）
3. 识别所需能力
4. 评估复杂度
5. 评估质量要求

输出: 结构化任务描述
```
